# Типы требований к ПО

## 📋 Классификация

### По уровню
| Тип | Описание | Пример |
|-----|----------|--------|
| BUSINESS | Бизнес-требования | Увеличить продажи на 20% |
| USER | Пользовательские требования | Пользователь может войти в систему |
| SYSTEM | Системные требования | Система должна обрабатывать 1000 запросов/сек |

### По назначению
| Тип | Описание | Пример |
|-----|----------|--------|
| FUNCTIONAL | Функциональные требования | Система должна отправлять email |
| NON_FUNCTIONAL | Нефункциональные требования | Время отклика < 2 секунд |
| TECHNICAL | Технические требования | Использовать PostgreSQL |
| UI | Требования к интерфейсу | Кнопка должна быть синей |

### По статусу
```cql
-- Статусы жизненного цикла требования
DRAFT      -- Черновик
REVIEW     -- На рассмотрении
APPROVED   -- Утверждено
REJECTED   -- Отклонено
IMPLEMENTED -- Реализовано
TESTED     -- Протестировано
VERIFIED   -- Проверено

### По приоритету
CRITICAL   -- Критическое (без него система не работает)
HIGH       -- Высокий (важно для бизнеса)
MEDIUM     -- Средний (желательно)
LOW        -- Низкий (можно отложить)
OPTIONAL   -- Опционально
